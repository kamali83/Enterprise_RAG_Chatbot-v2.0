ARCHITECTURE OVERVIEW
================================================================================

The project follows a RAG (Retrieval-Augmented Generation) architecture:

┌─────────────────────────────────────────────────────────────┐
│                     USER INTERFACE                          │
│              (frontend: HTML/JS/CSS files)                 │
└────────────────────┬────────────────────────────────────────┘
                     │
┌────────────────────▼────────────────────────────────────────┐
│                   WEB APPLICATION                           │
│                    (app.py)                                 │
│            • Routes & API endpoints                         │
│            • Request validation                             │
│            • Error handling                                 │
└────────┬──────────────────────────────────────────┬─────────┘
         │                                          │
         │                                          │
    ┌────▼──────────────────┐    ┌────────────────▼──────┐
    │   RETRIEVAL LAYER     │    │ GENERATION LAYER      │
    │   (retriever.py)      │    │ (generator.py)        │
    │                       │    │                       │
    │ • Query processing    │    │ • Prompt engineering  │
    │ • Vector search       │    │ • LLM integration     │
    │ • Doc ranking         │    │ • Response streaming  │
    └────┬──────────────────┘    └────────────────┬──────┘
         │                                        │
    ┌────▼──────────────────┐    ┌───────────────▼──────┐
    │   VECTOR DATABASE     │    │   LLM SERVICE        │
    │   (data/faiss_index/) │    │   (External API)     │
    │                       │    │                      │
    │ • index.faiss        │    │ • OpenAI / Similar   │
    │ • Embeddings         │    │ • Model selection    │
    └──────────────────────┘    └──────────────────────┘

         │ ◄─────────────────────────────────────► │
         │        DATA INGESTION PIPELINE         │
         │           (ingest.py)                  │
         │                                        │
    ┌────▼────────────────────────────────────────▼──────┐
    │        PERSISTENT STORAGE (Database Layer)        │
    │                                                    │
    │ • rag_saas.db (SQLite)                          │
    │ • users.json (User management)                  │
    │ • Caching layer (cashe.py)                      │
    └────────────────────────────────────────────────────┘

================================================================================
KEY DEPENDENCIES & INTEGRATIONS
================================================================================

• Language Model: LLM API (OpenAI/other)
• Vector Search: FAISS (Facebook AI Similarity Search)
• Database: SQLite (rag_saas.db)
• Web Framework: Flask or FastAPI
• Monitoring: Prometheus metrics
• Embeddings: Sentence Transformers / OpenAI Embeddings

================================================================================
CONFIGURATION & ENVIRONMENT
================================================================================

Configuration is loaded from:
1. .env file (environment variables)
2. config.py (Python configuration)
3. prometheus.yml (monitoring settings)

Key environment variables:
- Database URLs and credentials
- API keys for LLM services
- Vector store paths
- Logging levels

================================================================================
ENTRY POINTS & EXECUTION FLOW
================================================================================

1. Primary Entry Point: app.py
   - Initializes the web application
   - Loads configuration from config.py
   - Sets up routes and middleware

2. User Request Flow:
   frontend/ → app.py → retriever.py → generator.py → LLM

3. Data Processing Flow:
   ingest.py → data/docs/ → Data processing → FAISS index

4. Monitoring Flow:
   app.py → metrics.py → prometheus.yml → Prometheus

================================================================================
FILE SIZES & STORAGE
================================================================================

Critical Storage Elements:
• rag_saas.db: SQLite database (size varies)
• data/faiss_index/index.faiss: Vector index (size varies by document count)
• logs/app.log: Application logs (grows over time)

Cache Directories:
• __pycache__/: Python bytecode cache
• models/__pycache__/: Model module cache
• utils/__pycache__/: Utility module cache

================================================================================
DEVELOPMENT & MAINTENANCE
================================================================================

Required Packages:
- See requirements.txt for complete dependency list
- Install with: pip install -r requirements.txt

To Run the Application:
- Set environment variables in .env
- Execute: python app.py

To Ingest Documents:
- Use: python ingest.py
- Place documents in data/docs/

To Monitor Metrics:
- Access Prometheus metrics via configured endpoint
- Configuration in prometheus.yml
